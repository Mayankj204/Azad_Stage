1. User Role Hierarchy

Five-level access structure from national to district operations

1

Super Admin

National Level – Full Access

Complete access to the system including all modules and master settings. Highest level of control — can manage every aspect of the platform.

Dashboard FLP Management Assessments Training Batch Management Surveys Reports User Management ⚙ Master Settings
2

Admin

National Level – Operational Access

Access to all operational features of the system, but does not have access to the master settings.

Dashboard FLP Management Assessments Training Batch Management Surveys Reports User Management ⚙ Master Settings
3

State Lead

State Level

Access limited to state-level operations. Manages activities within their assigned state, oversees districts and centres under their jurisdiction.

State Dashboard FLP Management (State) Assessments (State) Training (State) Batch Management (State) Reports (State) User Management ⚙ Master Settings
4

District Lead

District Level

Manages operations at the district level. Oversees district-specific activities, FLPs, and users. Primary responsibility for batch creation and FLP allocation.

District Dashboard FLP Management (District) Batch Management Assessments (District) Reports (District)
5

Project Invigilator (PI)

District Level

Operates at the district level. Responsible for monitoring and managing project-related activities, field-level data collection, and FLP supervision.

District Dashboard FLP Management (District) Batch Management Assessments (District) Reports (District)

4. Batch Management Workflow

Step-by-step flow from batch creation to FLP allocation

1

Create Batch

A new batch is created within a state. Each batch has a unique state-wise batch code and is assigned to the current academic year. Only one active batch per state at a time.

Super Admin Admin State Lead District Lead ★ PI ★
District Leads and PIs are primarily responsible for batch creation in day-to-day operations.
2

Register FLPs

New FLPs (Field Level Persons) are registered into the system with their personal, demographic, and education details.

The Batch field is NOT mandatory during FLP registration. An FLP can be added without being assigned to a batch.
?

Batch Assignment Decision

At the time of FLP registration, the user decides whether to assign the FLP to a batch immediately or leave it unassigned for later allocation.

Assign Now
Select batch in FLP form → auto-linked
Assign Later
Leave blank → allocate via Batch Management
3

Batch Allocation (Batch Management Section)

District Leads or PIs access the Batch Management section to allocate unassigned FLPs into the appropriate batches. Multiple FLPs can be selected and assigned to a batch at once.

District Lead ★ PI ★ State Lead Admin
4

Training & Assessment

Once FLPs are assigned to batches, they proceed through Pre-Training Assessment → Training → Post-Training Assessment cycle. Batch grouping enables efficient tracking and reporting.

5. Batch Lifecycle

Annual batch cycle with automatic state-wise management

Create Batch

New batch created with unique state-wise code. One active batch per state at a time. Batch code format: {State}/{Year}/Batch {N}

Add & Allocate FLPs

FLPs are registered and allocated to the active batch either during registration or later via Batch Management.

Training Cycle

Batch undergoes Pre-Assessment → Training → Post-Assessment cycle. Progress tracked per batch.

Year Rollover

At new academic year, current batch becomes Inactive. New batch auto-created for the next year cycle.

6. Key Business Rules

Master Settings Access

Only the Super Admin can access Master Settings (States, Districts, Cities, Centres configuration). All other roles are restricted.

Data Scoping

State Leads see data for their state only. District Leads and PIs see data for their district only. Super Admin and Admin see all data.

One Active Batch Per State

Only one batch can be active per state at any time. When a new academic year begins, the previous batch is deactivated and a new one is created.

Batch Field is Optional

The Batch field in the FLP registration form is not mandatory. FLPs can be added without a batch and allocated later through Batch Management.

Automatic Yearly Rollover

Batches automatically change yearly. The system creates new batches for the current academic year and marks previous year's batches as Inactive.

Assessment Prerequisites

Post-Training Assessment can only be submitted after Pre-Training Assessment is completed. Once submitted, the FLP is removed from the Post-Assessment list.
